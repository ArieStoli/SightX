data storage¶

In this part I need to store the annotations - JSON files, in a database of some kind. I probably would have chosen MongoDB, a document database, for storage. It is a popular, versitile, NoSQL database that is easy to use and has a worldwide supporting community and documentation. It also has an extensive SDK for Python, which makes it easy to use.

I would also reccomend AWS's DynamoDB, which is also a NoSQL database (although being a key-value one). It will be more available than MongoDB and will automatically scale.

I currently don't have the resources to start these environments, so I will make a downscale dataset of a few movies for the analysis. Specifically, the annotation data from the SightX offices sub-directory. I have downloaded them to my computer and worked locally.

data import and conversion¶

In this part I will convert the data from a JSON format into the following Pandas dataframes:

  1. Movie_frames_df: stores the data for each frame, except for objects.
  2. Movie_metadata_df: stores the first row of each movie - information about the movie itself.
  3. Object_frames_df: stores the information about the objects in the movies, where and when they appear, and their different attributes.

object_col_names = []

def build_object_dict(objs, object_list, movie, frame):
    temp = {}
    for obj in objs:
        temp['movie'] = movie
        temp['frame'] = frame
        if 'attributes' in obj.keys():
            for k,v in obj['attributes'].items():
                temp[k] = v
                if k not in object_col_names:
                    object_col_names.append(k)
            
            obj.pop('attributes')
            for k,v in obj.items():
                temp[k] = v
                if k not in object_col_names:
                    object_col_names.append(k)
            object_list.append(temp)
        
    return object_list

Summary and suggestions¶

to summerize, due to the time and resource constraints and myself picking a dataset that doesn't contain huge amount of anomalies, would like to suggest some adjustments to enhance the dataset:

  • Combine more objects together to try and challange the models.
  • There are probably not enough data from these files to train a new model from scratch. It will need more.
  • Try to show an even amount of each object. There is a large bias towards nm 0.2 and nm 0.6 for example.
  • Try to include more frames without objects, so we could balance between situation where there is no data/objects to spot.
  • Clean the frames that are defined as 'ignore_frame' or 'attr_key_frame'. If the movie contains a large amount if those, so not include it in the content for the models.
  • If possible, change the shape of some of the objects from bBox to something more appropriate (maybe skeleton for presons), since the models are probably more adept in finding these objects with that type of shape, and because inconsistency is a bad thing.
